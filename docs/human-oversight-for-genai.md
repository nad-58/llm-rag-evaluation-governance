# Human Oversight for Generative AI

Human oversight defines when generated outputs can be used directly, when they require review, and when they must be escalated.

## Oversight triggers

- High-impact use case
- Low groundedness score
- Missing or weak source traceability
- Unsupported claims detected
- Ambiguous user request
- Out-of-scope request
- Conflicting source evidence
- Safety, privacy, clinical, legal, or financial sensitivity

## Oversight design questions

- Who reviews generated outputs?
- What evidence is shown to the reviewer?
- Can the reviewer edit, reject, or escalate the answer?
- Are review actions logged?
- Are recurring issues fed back into monitoring?
- Are users informed of limitations?

## Review expectation

For high-impact workflows, generated outputs should support human judgement rather than replace it. The oversight design should be proportional to the risk of the use case.
